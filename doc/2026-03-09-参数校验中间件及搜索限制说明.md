# 2026-03-09-新增参数校验中间件及搜索长度限制说明

## 1. 变更背景
为了提高系统安全性并优化数据库查询性能，防止无效或过短的模糊查询（如仅输入 1 个字符导致的性能开销），本项目引入了通用的参数校验机制。

## 2. 新增组件：`middleware/validate.js`
这是一个通用的校验中间件，允许在路由层灵活定义校验规则，支持对以下请求来源的校验：
- `req.body` (POST/PUT 请求体)
- `req.query` (GET 请求参数)
- `req.params` (URL 路径参数)

### 核心功能：
- **必填校验 (required)**：支持 `.trim()` 自动去除首尾空格，防止纯空格绕过校验。
- **长度校验 (min/max)**：限制字符串输入的最小及最大长度。
- **错误处理**：统一抛出 `ValidationError`，并返回标准的 400 错误响应。

## 3. 接口变更详情
本次更新对以下搜索接口应用了严格的校验规则：

### SN 搜索接口 (`GET /api/sn`)
- **变更点**：`keyword` 字段现在为**必填**。
- **限制**：最小长度必须为 **4 个字符**。
- **目的**：确保 SN 搜索具有足够的精确度，减少数据库全表扫描压力。

### 硬件搜索接口 (`GET /api/hw/cpu`)
- **变更点**：`keyword` 字段现在为**必填**。
- **限制**：最小长度必须为 **4 个字符**。

## 4. 后端错误返回示例
当校验不通过时，接口将返回如下格式：
```json
{
  "code": "VALIDATION_MIN_LENGTH",
  "message": "keyword 长度不能少于 4 个字符"
}
```
